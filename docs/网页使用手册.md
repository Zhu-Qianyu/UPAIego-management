## 1. 产品与适用对象

本系统在浏览器中提供以下能力（您实际能使用哪些，取决于**角色**与**是否已加入工作群**）：

- **在线设备**：在平台注册、展示心跳与状态、维护校准与备注、生成设备二维码。  
- **外部设备**：无法连接本站心跳的第三方设备，通过「甲方业务 + 设备简称」登记，获得**登记编号**与**贴签二维码**，由运维维护运行状态。  
- **工作群组**：平台管理员创建群、分发入群代码；成员申请入群、审批通过后在本群协作；群内**话题**沟通。  
- **场景业务**：采集排班（创建/发布/打卡）、大场景与小岗位维护；**甲方业务**与结算单价由**平台管理员**维护。  
- **悬赏令**：管理员发布工时池；数采执行员领取并完成。  
- **数采地图**：在地图查看设备位置与存储占用（当前版本可能显示「暂未上线」，见第 14 节）。  
- **管理台**：管理员配置各角色 **KPI**、发布**全员公告**，并查看**财务估算**看板。

## 2. 访问、浏览器与首屏

### 2.1 推荐环境

- 使用单位提供的 **HTTPS 站点地址**（具体域名由部署决定）。  
- 推荐浏览器：**Chrome、Edge、Firefox、Safari** 的较新版本。  
- 数采地图使用**高德地图 JS API**；功能开放后需在部署环境配置 Key（见第 14 节）。当前生产构建可能默认显示「地图功能暂未上线」占位页。  

### 2.2 首次打开

- **未登录**时，访问任意路径一般会跳转到 **`/auth`** 登录/注册页。  
- **已登录**时，若仍访问 `/auth`，会被重定向到工作台（按角色进入默认首页）。  
- 全局首次加载可能出现简短 **「加载中…」** 全屏提示，属正常。

## 3. 登录、注册与账号安全

### 3.1 登录

1. 在认证页点击 **「登录」**。  
2. 输入**手机号或邮箱**、**密码**，提交表单（老账号若无手机号绑定，请使用注册时的邮箱登录）。  
3. 若认证失败，页面会显示错误文案，请核对后重试。  

### 3.2 注册（若对您的环境开放）

1. 切换到 **「注册」**。  
2. 填写 **11 位中国大陆手机号**、密码；可选填**真实姓名**、**联系邮箱**。  
3. 选择注册方式之一：  
   - **注册为平台管理员**（勾选后无需群组号，不与其它职能兼任）；或  
   - **业务成员**：在 **设备运维员 / 场景业务员 / 数采执行员** 中**可多选**至少一种职能，并**必填群组号**（入群代码）。  
4. 填写群组号时，系统会预览对应工作群名称；无效代码无法提交。  
5. 提交成功后：  
   - **管理员**注册：通常可直接登录使用；  
   - **业务成员**：需等待平台管理员在 **群组管理** 审批通过后再使用群内业务功能。  

> **生产环境**：是否开放「注册为平台管理员」由单位策略决定；若无法注册，请联系管理员建号并分配职能。  

### 3.3 退出登录

- 顶栏右侧 **「退出」**：结束本会话，不删除账号。下次访问需重新登录。  

### 3.4 注销账号（永久删除）

- 顶栏 **「注销账号」** 打开弹窗。  
- 弹窗说明：操作**不可恢复**，将删除登录账号及关联数据，包括但不限于：  
  - 个人资料与角色  
  - 名下设备记录  
  - 您创建的采集排班与场景岗位数据  
  - 作为群主的工作群及群内话题、甲方需求、场景工位等（级联删除）  
  - 其它引用当前用户的数据  
- 另有一段提示：对象存储中的快照文件若仍存在，可能需在存储侧手动清理（一般由运维处理）。  
- 必须勾选 **「我已了解上述后果，确认要永久注销该账号」** 后，**「确认永久注销」** 按钮才可点。  
- 可点弹窗外部遮罩或 **「取消」** 关闭；处理中请勿重复点击。  
- 若报错提示数据库函数不存在等，需由运维执行仓库内迁移脚本（弹窗内会提示脚本文件名），属部署问题。

## 4. 角色与权限总览

登录后，顶栏显示**账号标识**（优先手机号/姓名，可点击进入 **个人信息**）与**职能标签**。若账号身兼多种非 admin 职能，顶栏会出现**职能下拉框**，用于切换当前工作台默认首页与 KPI 展示；**不**单独收窄您可访问的功能（权限为各职能的**并集**）。

| 职能 | 典型能力 |
|------|----------|
| **平台管理员** | 管理台（KPI、公告、**财务估算**）；创建**一个**工作群并审批入群、调整成员职能；设备管理含**全量设备**；场景业务三标签（含**甲方业务**）；发布**悬赏令**；数采地图入口 |
| **设备运维员** | 设备总览；**运维工作台**；设备管理（注册/搜索，无「全量设备」）；群组 |
| **场景业务员** | 场景业务（**采集排班**、**场景岗位**；**无**甲方业务 Tab）；群组 |
| **数采执行员** | **采集排班**执行视图（打卡）；设备总览（**只读**，**仅分配给本人**的设备）；悬赏令接单；**我的钱包**；数采地图入口；群组 |

**身兼数职示例**：同时拥有设备运维员 + 场景业务员时，侧栏会合并两类菜单项；切换顶栏职能只影响默认首页排序，仍可同时访问设备管理与场景业务。

**无权限页面**：直接访问不属于您职能并集的路由时，应用会重定向到合法首页，以线上行为为准。

## 5. 界面布局与通用交互

### 5.1 左侧导航栏

- **菜单项**：因职能而异；身兼数职时为各职能菜单的**并集**（同一路径不重复）。  
- **站点标题**：点击后进入当前**工作台职能**对应的默认首页：  
  - 平台管理员 → 管理台  
  - 设备运维员 → 设备总览（`/`）  
  - 场景业务员 → 场景业务（`/scene`）  
  - 数采执行员 → 数采地图（`/map`；若地图暂未上线则仍进入该路径）  
- **收起侧栏**（仅桌面端显示箭头按钮）：侧栏变窄，菜单只显示每条首字缩写；鼠标悬停可看完整名称。  
- **手机端**：顶栏左侧 **「菜单」** 打开侧栏；点击灰色遮罩、切换路由、或按 **Esc** 可关闭侧栏。  

### 5.2 顶栏

- **账号链接**（进入 **个人信息** `/profile`）、**职能标签或下拉框**、**注销账号**、**退出**。  
- 鼠标悬停职能区域可查看各职能职责说明。  

### 5.3 主内容区顶部提示（可能出现）

1. **群组状态横幅**（蓝色或琥珀色）  
   - **待审批**：入群申请已提交，尚未通过。  
   - **未入群**：若您是管理员，文案会引导去 **群组管理** 建群或去 **群组** 用代码加入；非管理员则引导去 **群组** 申请入群。  

2. **KPI 完成进度**（当管理员为您配置了**当前有效期内**的 KPI 时；身兼数职时按顶栏所选**工作台职能**匹配对应 KPI）  
   - 展示指标名称、当前值、目标、进度条与考核时间范围。  

3. **管理员公告**（琥珀色卡片，多条）  
   - 展示管理员近期发布的全员留言（标题、正文、时间）。  

### 5.4 刷新与加载

- 许多列表页在后台重新拉取数据时，顶部可能出现细条状的 **刷新中** 提示（`RefreshStrip`）。  
- 大段数据加载中会显示 **转圈加载**，请稍候。  

### 5.5 个人信息 `/profile`

- 顶栏账号链接进入 **个人信息** 页。  
- 可维护 **真实姓名**、**显示名**、**手机号**、**联系邮箱**（不影响以手机号注册账号的登录方式）。  
- 若以邮箱注册的老账号，可在此修改 **登录邮箱** 与 **密码**；纯手机号账号登录请继续使用手机号。  
- 页内只读展示当前 **平台职能** 列表（职能变更需联系管理员在 **群组管理** 调整）。

## 6. 工作群组与群组管理

### 6.1 基本规则

- **只有平台管理员**能在 **群组管理**（`/group/manage`）中**创建一个**工作群组（每位管理员作为群主通常仅允许持有一个此类群组，具体以创建时的错误提示为准）。  
- **其它角色不能自建群**，只能使用管理员提供的 **入群代码** 在 **群组**（`/group`）页提交申请。  
- 群组页顶部有 **「群组」** 与 **「群组管理」** 两个标签；后者**仅管理员可见**。  

### 6.2 群组页 `/group` — 未入群

1. 阅读说明：工作群仅由平台管理员创建。  
2. 在 **入群代码** 输入框填入管理员提供的代码（界面示例形如 `A1B2C3D4`；实际格式以单位约定为准）。  
3. 点击 **「提交申请」**。  
4. 成功后会提示申请已提交，需等待审批；可再次进入本页点击 **「刷新状态」** 查看是否已通过。  

### 6.3 群组页 — 申请审核中

- 若您已提交申请但尚未通过：页面标题区域为 **「入群申请处理中」**，说明审批由**群主或平台管理员**处理。  
- 使用 **「刷新状态」** 手动更新。  

### 6.4 群组页 — 已加入

**群信息**

- 显示当前激活群的**显示名称**及简短说明。  

**成员列表**

- 列：**成员**（优先显示资料中的显示名，否则为申请邮箱或用户 ID 缩写）、**平台职能**（如设备运维员；身兼数职时以顿号连接）、**成员状态**：  
  - **已加入**：可正常使用群内业务数据。  
  - **待审核**：尚未激活。  
  - **已拒绝**：未通过。  
- 标注 **（我）** 表示当前登录用户；**（群主）** 表示群主。  
- **移出**：仅 **平台管理员** 或 **群主** 可对部分成员显示；不能移出自己、群主、已拒绝成员；操作前有浏览器级 **确认** 对话框。  

**话题**

- **发布话题**：必填 **标题**；**正文** 可选；提交 **「发布」**。  
- **话题列表**：展示标题、正文、创建时间。  
- **删除**：**话题发布者本人**或 **平台管理员** 可对单条话题删除；删除前有确认。  

### 6.5 群组管理 `/group/manage`（仅管理员）

**尚未创建群组**

- 表单字段：  
  - **群组显示名称**（必填）  
  - **自定义入群代码**（可选；不填则系统自动生成 **8 位**；若填写，输入会自动转为大写）  
- 提交 **「创建群组」**。  
- 若违反「每位管理员仅一个群」等规则，页面红色区域会显示错误原因。  

**已拥有群组**

- 展示群名称与 **入群代码**（大号等宽字体，便于复制给同事）。  
- **待审批入群**列表：显示申请人信息；审批时可勾选/调整其**业务职能**（设备运维员、场景业务员、数采执行员可多选）；每条可 **通过** 或 **拒绝**。  
- **已入群成员 · 调整职能**：对已激活成员可随时修改职能组合（多选）；**平台管理员账号**不在此列表修改。  
- 可使用 **「刷新」** 更新列表。

## 7. 设备总览（运维员 / 执行员）

路径：**设备总览**（`/`）。**设备运维员**默认首页；**数采执行员**也可访问，但为**只读模式**，且**仅展示分配给该执行员的设备**（见下）。**平台管理员**在「全量设备」标签内打开时标题为 **「全平台设备总览」**（见第 8 节）。

### 7.0 数采执行员的数据范围（重要）

执行员在设备总览中**不能**查看本群全部设备，只能看到**分配给自己**的条目，来源包括：

1. **运维批量/单台分配**：设备管理中对离线设备执行「分配给执行员」后，对应登记编号会出现在执行员总览。  
2. **采集排班发布**：为本执行员发布排班时绑定的离线设备登记编号（已发布或已关闭的排班）。  
3. **联网设备**：仅当已通过分配关系绑定到该执行员时可见（与离线设备共用分配表）。  

未分配给您的设备不会出现在列表中；直接打开他人设备详情链接也会提示未找到或无权。

### 7.1 页面说明文案（重要）

- 在线设备列表会**优先展示**需要关注的项：如心跳离线、非「正常」业务状态、待校准或需重校准等（排序由系统自动处理）。  
- **外部设备**表格会优先展示 **返厂维修**、**异常**，其次为正常。  
- 当列表范围为「全平台」相关视图时，文案会说明：**仅展示当前工作群内成员名下的设备**（管理员视角）；**数采执行员**不受此条约束，仍只看**本人被分配**的设备。  

### 7.2 筛选与统计

- 两个下拉框：  
  - **设备业务状态**：全部状态 / 正常 / 未激活 / 维护中 / 已退役  
  - **校准状态**：全部校准状态 / 待校准 / 已校准 / 需重校准  
- 更改筛选会回到**第 1 页**。  
- 右上角统计：**在线**设备台数；若有外部设备数据，会显示 **· 外部设备 N 台**。  

### 7.3 在线设备表格

- 列：**设备名称**、**设备 ID**、**序列号**、**状态**、**校准**、**注册时间**、操作 **「查看」**。  
- **状态**列旁有 **刷新在线状态** 按钮：点击后会重新拉取列表并刷新连通性显示（与心跳判断逻辑相关）。  
- **表头排序**：除「状态」列与刷新按钮组合外，各列可点击表头切换升序/降序；状态列也可点击文字部分排序。  
- 默认排序在「在线关注优先级」一致时，按当前排序列的字母序规则排列。  
- **「查看」** 进入在线设备详情 `/devices/{设备ID}`。  

### 7.4 外部设备表格

- 区块标题：**外部设备（无法连接本站）**。  
- 列：**设备类型**（由系统格式化的「甲方 + 简称」类标签）、**登记编号**、**内部 ID**、**状态**（正常 / 异常（据人员反馈）/ 返厂维修 等界面文案）、**登记时间**、**「打开」**。  
- **「打开」** 进入 `/devices/manual/{登记编号}`。  

### 7.5 分页

- 每页 **20** 台在线设备（固定页大小）。  
- 底部 **上一页** / **下一页** 与 **第 x / y 页**；仅当总页数大于 1 时显示。  

### 7.6 空状态

- 若在线设备与外部设备均为空：提示 **「暂无设备数据」**（执行员只读模式下为 **「暂无分配给您的设备」**），运维员可见链接 **「去注册第一台设备」**。

## 8. 设备管理：注册、外部设备、搜索、全量

路径：**设备管理**（`/devices/manage`）。**管理员**与 **设备运维员** 可访问。

页面顶部有一段总述：**注册设备** 含在线注册与 **外部设备** 登记；**搜索** 支持设备 ID、外部登记编号等；管理员另有 **全量在线设备**。

### 8.1 标签切换

| 标签 | URL 习惯 | 说明 |
|------|-----------|------|
| 注册设备 | 默认无 `tab` 参数 | 在线注册 + 页面向下的外部设备区块 |
| 搜索设备 | `?tab=search` | 独立搜索表单与结果 |
| 全量设备 | `?tab=fleet` | **仅管理员**可见；非管理员若强行带参数会被纠正回默认标签 |

### 8.2 注册设备 — 在线设备

1. **序列号（可选）**：可填写设备 CPU 序列号等；页面提示可通过单位提供的 CLI 获取。  
2. 点击 **「确认注册设备」**。  
3. 成功后：  
   - 显示绿色对勾与 **设备注册成功**  
   - 展示 **可读名称** 与 **设备 ID**（等宽小字）  
   - 展示 **二维码**；说明请下载打印贴到设备上  
   - 按钮：**下载二维码**、**查看设备**（进详情页）、**继续注册**（清空表单再注册一台）  

> **管理员**注册的在线设备归属列表范围与 **运维员** 可能不同（后台 `listScope` 区分 fleet / own），以单位数据策略为准。  

### 8.3 注册设备 — 外部设备（`ManualDevicesTab`）

**前提**

- 必须先 **加入已审批的工作群组**；否则会提示先加群，无法登记。  
- 登记时需关联 **甲方业务**：下拉列表来自当前群的 **甲方业务** 数据。若下拉为空，页面会提示：**请平台管理员在「场景业务 → 甲方业务」中添加**。  

**新增一条外部设备**

1. **关联甲方业务**：在下拉框选择一条（展示为公司名及设备类型括号说明）。  
2. **设备简称**（必填）：如「产线 A 控制柜」。  
3. 点击 **「生成登记编号与二维码」**。  
4. 成功后列表 **「本群已登记」** 会出现新卡片；可点 **「刷新」** 更新列表。  

**每条外部设备卡片**

- 左侧：**二维码**（扫码打开与本贴签相同的 **外部设备详情页** URL）。  
- 文案：**登记编号（贴签）**、**内部 ID**。  
- **设备状态** 下拉：**正常** / **异常（据人员反馈）** / **返厂维修**；修改后点 **「保存状态」**（仅状态变更时可点）。  
- **删除外部设备**：链接式按钮，确认后删除登记（登记编号作废）。  

### 8.4 搜索设备

1. 在输入框填写关键词（占位提示包含：设备 ID、名称、序列号、备注，或外部设备登记编号 / UUID / 设备简称等）。  
2. 点击 **「搜索」** 或表单提交。  
3. 结果区：  
   - 一行小字：**在线设备 N 条；外部设备 M 条**  
   - **在线设备表**：列含 **来源**（固定显示「在线」）、设备名称、设备 ID、序列号、**状态**（表头旁带刷新按钮）、校准、备注、**查看**  
   - **外部设备表**（若有）：设备类型、登记编号、内部 ID、状态、**打开**  
4. 无结果时提示 **「没有匹配的设备或外部设备。」**  
5. **状态列旁的刷新**：在不改关键词的情况下重新查询并刷新在线状态显示。  

### 8.5 全量设备（仅管理员）

- 与设备总览使用相同 **Dashboard** 组件，但数据范围为 **fleet（全量）** 视角。  
- 仍受文案中所述「工作群内成员名下」等数据范围约束（与后端策略一致）。

## 9. 在线设备详情页

路径：`/devices/{设备ID}`（ID 为 URL 编码后的设备 ID）。

### 9.1 返回与权限

- 左上角 **「← 返回设备总览」** 回到 `/`。  
- 若无权或设备不存在：提示 **「未找到该设备，或你没有访问权限」**。  

### 9.2 信息卡（左侧主区域）

- **标题**：可读名称；副标题为 **设备 ID**。  
- **状态角标** + **刷新在线状态** 按钮（与总览类似，针对单台刷新）。  
- **只读字段网格**：  
  - 序列号  
  - 注册时间  
  - 最近在线  
  - 校准状态（徽章）  
  - 固件版本  
  - **实时 CPU 占用率**（若板端上报了 runtime 数据则显示百分比，否则 `-`）  
  - **最新图片帧时间**  
- **设备实时图片帧（每分钟更新）**：若有 base64 帧则显示图片；否则 **「暂未收到图片帧」**。  
- **备注**：若设备有备注则只读展示一段。  

### 9.3 编辑设备信息

- **状态** 下拉：正常 / 未激活 / 维护中 / 已退役  
- **校准状态** 下拉：待校准 / 已校准 / 需重校准  
- **固件版本** 文本框  
- **备注** 多行文本  
- **「保存修改」**：成功后短暂显示绿色 **「修改已保存」**。  
- 错误时显示红色错误文案。  

### 9.4 危险操作

- **「设为退役」**：浏览器确认文案为 **「确认将该设备设为已退役吗？」**；确认后调用删除/退役逻辑并跳回设备总览。  

### 9.5 右侧二维码卡

- 展示设备二维码、**「下载二维码」**、说明文字 **「扫码即可快速识别设备信息」**。  

### 9.6 自动刷新

- 页面约每 **5 秒** 静默拉取一次设备详情，便于观察在线与 runtime 变化（网络失败时保留当前界面，不刷屏报错）。

## 10. 外部设备详情（按登记编号）

### 10.1 登记编号格式

- 标题 **「外部设备」**；说明无法连接本站心跳、由运维据反馈维护状态。  
- 展示设备类型标签（格式化后的甲方 + 简称）。  
- **设备状态** 下拉与 **「保存状态」**（同注册页列表逻辑）。  
- 展示 **登记编号**；若有则展示 **与贴签相同的链接二维码**。  
- 底部 **「返回设备管理」**。

## 11. 管理台（管理员）

路径：**管理台**（`/admin`）。

### 11.1 KPI 设置

**规则摘要**

- 适用角色仅限：**设备运维员**、**场景业务员**、**数采执行员**。  
- 每个角色对应**固定指标**（不可自定义指标名称）：  
  - 设备运维员 → **设备完好率**（目标为百分比数值，如 95 表示 95%；单位可填 `%`）  
  - 场景业务员 → **场景数**（目标为场景个数；单位可填「个」等）  
  - 数采执行员 → **数据量**（目标为与单位一致的累计用量，如单位填 MB 则与已用存储 MB 对齐）  
- **目标值**：必填，且须为 **大于 0** 的数字。  
- **单位**、**备注**：可选。  
- **考核开始 / 考核结束**：可选 `datetime-local`；若两者都填，开始不能晚于结束。  
- 提交按钮：**「添加 / 更新 KPI」**（与后端 upsert 行为一致，同角色同指标会更新）。  

**已配置的 KPI 列表**

- 每条显示：指标标题、目标角色徽章、目标值与单位、备注、起止时间、**「当前不在有效期内」** 提示（若适用）、更新时间。  
- **「删除」**：需确认 **「删除该 KPI？」**。  

### 11.2 全员留言（公告）

- **发布公告**：必填 **标题**、**正文**；按钮 **「发布公告」**。  
- 发布后，所有职能用户登录后在主内容区顶部 **公告卡片** 区域可见（列表拉取最近若干条，与后台 `listAdminMessages` 条数一致）。  
- **近期公告**：管理台内可滚动查看历史公告列表。  

### 11.3 财务估算看板

在已创建工作群的前提下，管理台下方展示本群经营数据摘要（需已配置甲方结算单价时，收入侧才显示具体金额，否则为 **—**）：

- **汇总卡片**：数采登记工时、执行员结算成本、甲方收入估算、净利润估算等。  
- **按甲方拆分**：各甲方公司对应的小时量、设备数、结算单价（仅管理员可见）、估算收入。  
- 说明文案：**成本**为执行员结算支出；**净利润估算** ≈ 甲方单价 × 数采登记工时 − 成本（场景累计工时计入收入侧，暂不按周月拆分）。  

> **甲方结算单价**仅在 **场景业务 → 甲方业务** 中由平台管理员填写；其它职能在界面与 API 中均不可见该字段。

## 12. 场景业务 / 采集排班

路径：**场景业务**（`/scene`；数采执行员侧栏也可能显示为 **采集排班**，同一路径）。**必须先有已审批入群的工作群**，否则整页引导去 **群组** 申请入群。

页面 URL 可用 `?tab=` 切换标签：`tasks`（采集排班）、`demands`（甲方业务，仅管理员）、`stations`（场景岗位）。非管理员访问 `?tab=demands` 会自动回到采集排班。

### 12.1 数采执行员（执行视图）

- 页面大标题为 **「采集排班」**（无标签栏）。  
- 副标题：查看已发布排班、本批设备登记编号，并按时上下班打卡。  
- 仅展示 **已发布** 的排班卡片；每条显示场景岗位、计划/发布时间、**本批设备编号**（贴签 hex）。  
- **上班打卡 / 下班打卡**：在已发布排班卡片内操作；已上班时显示上班时间。  
- **无**创建、发布、删除、关闭排班入口；**无**甲方业务、场景岗位维护 Tab。  

### 12.2 场景业务员与平台管理员（管理视图）

- 页面大标题为 **「场景业务」**。  
- **场景业务员**可见两个标签：**采集排班**、**场景岗位**。  
- **平台管理员**额外可见第三标签：**甲方业务**。  

#### 采集排班（Tab `tasks`）

面向管理员与场景业务员的手动排班流程（取代旧版「场景任务」主流程）：

1. **创建排班草稿**：选择 **场景岗位**（小岗位）、**数采执行员**、**设备数量**；可选计划上/下班时间与备注。  
2. 提交 **「创建排班草稿」**；若本群尚无小岗位或执行员，页面会提示先去场景岗位添加或等待成员入群。  
3. 创建成功后可确认 **立即发布并分配设备**；发布时系统从本群可用离线设备中匀出一批 **登记编号**。  
4. **草稿**状态：可 **发布并排设备**、单条或批量 **删除草稿**。  
5. **已发布**：执行员可见并打卡；管理员/业务员可 **关闭排班**（关闭后执行员不可继续打卡，设备释放）。  
6. **已关闭**：只读展示历史。  

列表支持紧凑/卡片视图切换；草稿可多选批量删除。

#### 甲方业务（Tab `demands`，**仅平台管理员**）

维护本群甲方数采需求，供外部设备登记、排班与财务估算引用：

- **字段**：甲方公司、设备类型、**设备快照**（必填上传）、需求总小时量（可勾选「无限」）、**各大场景批复小时数**（正整数，按每个大场景分别填写）、**甲方结算单价**（元/小时，可选，**仅管理员可见**）、其它说明。  
- 若本群**尚无大场景**：可先保存甲方业务，待 **场景岗位** 中新增大场景后再编辑补全批复小时。  
- 列表卡片展示设备类型、批复小时汇总（如 `6~12h（2 个大场景）`）等；支持编辑、单条/批量删除、**打印列表**（含快照图与各场景批复；含单价列）。  
- **场景业务员、数采执行员**无此 Tab；即使直接访问也会被重定向或看到无权限提示。  

#### 场景岗位（Tab `stations`）

两级结构：**大场景** → **小岗位**。

**大场景**

- **添加大场景**：名称（必填）、说明、**全景图**（必填上传）、联系人、电话、**省市区**（必填）、详细地址。  
- 列表可编辑、单条/批量删除、**多选打印**（可选打印字段）。  
- **删除大场景**：会级联删除其下小岗位与**草稿**排班；若仍存在**已发布或已关闭**的采集排班，会提示先处理后再删。  

**小岗位**（在大场景卡片内维护）

- **添加工序 / 小岗位**：工序名称、工位说明、**工位快照**上传等。  
- 可编辑、删除；删除前确认。  
- 排班创建时的「场景岗位」下拉即来自此处。  

页面说明：先添加大场景（含全景图与联系人），再在各 **大场景卡片内** 维护下属小岗位。

### 12.3 场景大类（系统默认）

甲方业务保存时系统会附带默认场景大类标签（**工业 / 家庭 / 特种**），用于后台分类与同步逻辑；日常操作以 **大场景批复小时** 为准，无需在表单中逐项勾选大类。

## 14. 数采地图

路径：**数采地图**（`/map`）。**平台管理员**与 **数采执行员** 侧栏可见。

### 14.0 当前版本说明

若线上显示 **「地图功能暂未上线」** 占位页，表示该功能在当次部署中尚未开放；下列 14.1~14.6 为功能开放后的预期行为。开放需运维在代码/环境中启用地图开关并配置高德 Key 后重新构建部署。

### 14.1 布局

- 左侧（宽屏为 2/3 宽）：**高德地图**（JS API 2.0）。  
- 右侧：设备卡片列表（可滚动，最大高度与地图大致对齐）。  

### 14.2 地图行为

- 仅展示在数据库中具备 **`map_lat` / `map_lng`** 的设备；无坐标设备仅在列表中提示 **「未标定地图坐标」**。  
- 有坐标时地图会自动 **缩放到包含所有点** 的视野（带边距与最大缩放限制）。  
- 点击地图标记：弹出气泡，含设备名、**存储百分比**、设备 ID。  

### 14.3 设备卡片

- 展示名称、设备 ID、存储进度条、**已用 MB / 容量 MB**（容量缺省按界面默认展示）。  
- 当存储比例 **≥ 约 95%**：卡片高亮，显示 **「可收菜」**；**「回收数据」** 可点。  
- 未满时按钮文案为 **「存储未满」** 且禁用。  
- 回收进行中该卡按钮会显示 **「处理中…」**。  

### 14.4 刷新策略

- **「刷新」** 按钮：立即拉取列表。  
- 进入页面后约每 **30 秒** 自动重新加载设备列表。  

### 14.5 地图配置（运维）

- 在 `frontend/.env` 配置 **`VITE_AMAP_KEY`**（Web 端 Key）与 **`VITE_AMAP_SECURITY_CODE`**（安全密钥）；高德开放平台需绑定本站域名（含 `localhost` 与生产域名）。  
- 设备经纬度默认为 **GPS（WGS84）**，页面会自动转换为高德坐标；若库中已是 GCJ-02，可设 **`VITE_MAP_COORD_SOURCE=gcj02`**。  
- 未配置 Key 或 Key 无效时，地图区域显示配置说明或红色错误提示。  

### 14.6 数据范围

- 页面说明：列表数据来自对 **fleet** 范围的设备列举接口；业务上强调与 **当前工作群内设备** 等业务规则一致，若地图无点请核对入群与坐标维护。

## 17. 术语与状态说明

### 17.1 在线设备 — 业务状态（`status`）

- **正常（active）**  
- **未激活（inactive）**  
- **维护中（maintenance）**  
- **已退役（retired）**  

界面上的 **在线/离线** 与心跳相关显示，可能叠加短暂迟滞（防抖/迟滞逻辑），以 **刷新在线状态** 后为准。  

### 17.2 在线设备 — 校准状态（`calibration_status`）

- **待校准（pending）**  
- **已校准（calibrated）**  
- **需重校准（needs_recalibration）**  

### 17.3 外部设备 — 运行状态（`external_status`）

- **正常**  
- **异常（据人员反馈）**  
- **返厂维修**

## 18. 常见问题排查

1. **登录后黄页「未检测到账号角色」**  
   - 联系管理员在 `profiles` 中为您绑定 `id`（用户 UUID）与 `roles`；或使用页面上的 SQL 模板；完成后点 **同步并重试**。  

2. **场景业务整页让我去入群**  
   - 说明当前没有已激活的群成员身份；请到 **群组** 用入群代码申请，或联系管理员在 **群组管理** 通过审批。  

3. **外部设备登记里没有「甲方业务」可选**  
   - 请 **平台管理员** 在 **场景业务 → 甲方业务** 中先创建甲方需求数据。  

4. **我是场景业务员，为什么看不到「甲方业务」？**  
   - 该 Tab **仅平台管理员**可见；请联系管理员维护甲方公司与批复小时、结算单价。  

5. **数采地图显示「暂未上线」或没有点**  
   - 「暂未上线」表示功能未在该版本开放；若已开放仍无点，请确认设备是否维护经纬度、账号是否在正确工作群、高德 Key 是否有效。  

6. **搜索不到设备**  
   - 检查关键词是否含多余空格；管理员与运维员的**搜索范围**可能不同（fleet vs own）；外部设备是否与当前群关联。  
   - **数采执行员**在总览/搜索中仅能命中**已分配给自己**的设备；未分配的设备不会出现。  

7. **KPI 条不显示**  
   - 需管理员配置**当前时间在有效期内**的 KPI；身兼数职时请确认顶栏 **工作台职能** 与 KPI 角色一致。  

8. **注销账号失败**  
   - 看弹窗错误信息；若提示数据库函数缺失，需运维执行仓库内用户注销相关 SQL 迁移。  

9. **注销后同一邮箱再注册提示 `email rate limit exceeded`**  
   - 这是 **Supabase 认证邮件发送频率限制**，不是网站业务拒绝注册。注销会删账号，但再次 `signUp` 仍可能触发「注册确认邮件」，与同一邮箱短时间内的注册/确认/找回密码等邮件合计超限（免费版常见为每小时仅少量邮件）。  
   - **处理**：等待约 1 小时后再试；或请运维在 Supabase → **Authentication → Rate Limits** 调高限额；开发/内网环境可在 **Authentication → Providers → Email** 关闭 **Confirm email**，则注册后通常无需再等确认邮件、也不易触发限流。  
   - 若未开邮箱确认，注册成功后应可直接登录；若仍失败，检查是否启用了确认邮件且尚未点击邮件中的确认链接。
